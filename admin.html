Configuração do Supabase

Execute os passos abaixo no seu projeto Supabase para ativar o painel admin com autenticação segura.

1 — Criar usuário admin

1Acesse supabase.com/dashboard
2Vá em Authentication → Users → Add user
3Crie com seu e-mail e uma senha segura
4Use esses dados para fazer login neste painel

2 — Tabela de produtos + políticas seguras

No SQL Editor do Supabase, execute:

-- Tabela de produtos CREATE TABLE IF NOT EXISTS products ( id uuid DEFAULT gen_random_uuid() PRIMARY KEY, name text NOT NULL, price text NOT NULL, description text DEFAULT '', category text NOT NULL, image_url text DEFAULT '', badge text DEFAULT '', created_at timestamptz DEFAULT now() ); -- Habilitar RLS ALTER TABLE products ENABLE ROW LEVEL SECURITY; -- Qualquer visitante pode LER os produtos (vitrine pública) CREATE POLICY "public_read" ON products FOR SELECT TO public USING (true); -- Apenas usuários autenticados (admin logado) podem ESCREVER CREATE POLICY "auth_write" ON products FOR ALL TO authenticated USING (true) WITH CHECK (true);

3 — Bucket de imagens seguro

No SQL Editor, execute também:

-- Criar bucket público (leitura pública, upload restrito) INSERT INTO storage.buckets (id, name, public) VALUES ('produtos', 'produtos', true) ON CONFLICT (id) DO UPDATE SET public = true; -- Apenas autenticados podem fazer upload CREATE POLICY "auth_upload" ON storage.objects FOR INSERT TO authenticated WITH CHECK (bucket_id = 'produtos'); -- Apenas autenticados podem deletar imagens CREATE POLICY "auth_delete" ON storage.objects FOR DELETE TO authenticated USING (bucket_id = 'produtos'); -- Qualquer visitante pode ver as imagens (URL pública) CREATE POLICY "public_images" ON storage.objects FOR SELECT TO public USING (bucket_id = 'produtos');

Segurança

Com essas políticas, somente usuários autenticados no Supabase conseguem adicionar, editar ou excluir produtos e fazer upload de imagens. Visitantes do site só conseguem visualizar.
